# Cosy Domain Provider — Frontend

React SPA for claiming and managing `cosy-hosting.net` subdomains. Pixel-art / retro aesthetic matching the main Cosy application. Backend counterpart: `../cosy-domain-provider` (Spring Boot).

See `specs/` for detailed page designs and the design-system reference.

---

## Commands

```bash
npm run dev        # Dev server at localhost:5173 (proxies /api → http://127.0.0.1:8080)
npm run build      # TypeScript check + Vite production build
npm run lint       # Biome lint + format check
npm run lint:fix   # Biome auto-fix
npm run format     # Biome format only
npm run gen:api    # Regenerate API client from running backend's OpenAPI spec
npm run preview    # Preview production build locally
```

`gen:api` curls `http://localhost:8080/v3/api-docs` into `openapi-domain-provider.json`, then runs Orval to regenerate `src/api/generated/`. The backend must be running first.

## Routing (TanStack Router)

Routes live in `src/routes/` as `.tsx` files. The router plugin auto-generates `src/routeTree.gen.ts` on dev server start — never edit it manually.

**Current routes:**
| File | Path | Auth |
|---|---|---|
| `__root.tsx` | layout wrapper | — |
| `index.tsx` | `/` | public |
| `login.tsx` | `/login` | redirect to `/dashboard` if logged in |
| `register.tsx` | `/register` | redirect to `/dashboard` if logged in |
| `dashboard.tsx` | `/dashboard` | required |
| `domain.$domainId.tsx` | `/domain/:domainId` | required |

**Auth guard pattern** (in `beforeLoad`):
```ts
const token = store.getState().auth.identityToken;
if (!token) throw redirect({ to: '/login' });
```

**Redirect-if-logged-in pattern** (login/register):
```ts
const token = store.getState().auth.identityToken;
if (token) throw redirect({ to: '/dashboard' });
```

Router is configured with `defaultPreload: 'intent'`.

## State management (Redux)

```
store.auth = {
  identityToken: string | null
  user: { username, email, subject, issuedAt, expiresAt, claims } | null
  bootstrapped: boolean     // false until auth check completes on app start
  state: 'idle' | 'loading' | 'failed'
}

store.subdomains = {
  items: SubdomainDto[]
  state: 'idle' | 'loading' | 'failed'
  errorMessage: string | null
}
```

**Key actions:**
- `setIdentity({ token, user })` — store token + parsed user
- `clearIdentity()` — on logout
- `markBootstrapped()` — auth check done (show app even if not logged in)
- `setSubdomains(items)` — replace subdomain list
- `upsertSubdomain(item)` — insert or update single subdomain
- `removeSubdomain(uuid)` — remove from list

Components read state via `useAppSelector`. Never mutate state directly outside a slice.

## Hook patterns

### `useDataLoading` — read operations

```ts
const { bootstrapAuth, loadSubdomains, loadSubdomainByUuid } = useDataLoading();

// Called once on app start (inside AuthProvider)
await bootstrapAuth();  // GET /api/v1/auth/token → dispatches setIdentity + markBootstrapped

// Called when entering dashboard
await loadSubdomains(); // GET /api/v1/subdomain → dispatches setSubdomains

// Called when entering domain detail page
await loadSubdomainByUuid(uuid); // GET /api/v1/subdomain/:uuid → dispatches upsertSubdomain
```

### `useDataInteractions` — mutations

```ts
const {
  loginUser, registerUser, logoutUser,
  createSubdomain, updateSubdomain, deleteSubdomain,
  refreshIdentityToken
} = useDataInteractions();

await loginUser({ username, password });     // dispatches setIdentity, navigates /dashboard
await registerUser({ username, email, password }); // navigates /login
await logoutUser();                           // dispatches clearIdentity, navigates /login
await createSubdomain({ label, targetIp });  // dispatches upsertSubdomain
await updateSubdomain(uuid, { targetIp });   // dispatches upsertSubdomain
await deleteSubdomain(uuid);                 // dispatches removeSubdomain
```

### `useAuthInformation` — auth state selectors

```ts
const {
  isUserLoggedIn,    // boolean
  isBootstrapped,    // boolean — false during initial auth check
  userName,          // string | null
  userEmail,         // string | null
  tokenExpiresAt,    // Date | null
  logoutUser,        // () => void
} = useAuthInformation();
```

## API codegen (Orval)

`npm run gen:api` curls the backend OpenAPI spec and runs Orval:
- Output: `src/api/generated/domain-provider-api.ts` (functions + hooks)
- Output: `src/api/generated/model/` (TypeScript DTOs)

**Do not edit generated files.** Custom API logic goes in the hooks (`useDataLoading`, `useDataInteractions`), not in the generated code.

Orval generates three exports per endpoint — example:
```ts
function listMySubdomains(options?)               // raw axios call
function getListMySubdomainsMutationOptions()     // UseMutationOptions factory
function useListMySubdomains(options?)            // React Query hook
```

## i18n

Translation keys live in `src/i18n/resources.ts` as a TypeScript object (not JSON). **Always add to both `en` and `de`.**

```ts
// Usage
const { t } = useTranslation();
t('dashboard.title')
```

Language stored in `localStorage` under key `'cosy-language'`.

## Adding a new page

1. Create `src/pages/my-page.tsx` — named export `export function MyPage()`
2. Create `src/routes/my-page.tsx` — use `createFileRoute('/my-page')()`
3. Add `beforeLoad` auth guard if required
4. Add i18n keys to `resources.ts` (both `en` and `de`)
5. `routeTree.gen.ts` regenerates automatically on next dev server start
